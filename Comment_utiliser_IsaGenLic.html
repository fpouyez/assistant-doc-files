Comment utiliser IsaGenLic ou GenLic

Présentation générale

Ce document va décrire les différentes fonctionnalités disponibles dans l’application IsaGenLic.

Pour démarrer l’application, il faut lancer « IP.Licence.GenLic.Application.exe »

La dernière version de l’application (sans paramétrage de défini) est présente sur P:\Outils\Utilitaires\IsagenLic\Main. Il suffit de la copier sur votre PC pour pouvoir l’utiliser.

Paramétrer l’application

Avant de pouvoir créer ou ouvrir une licence, il faut paramétrer différents éléments via le groupe « Configuration » du ruban « Accueil ».

Paramétrage des sources de données

Pour pouvoir ouvrir une licence depuis une base de données, enregistrer une licence directement dans une base ou installer une licence en base, il faut configurer les sources de données disponibles via le bouton « Gestion des sources de données ».

Une source de donnée est décrite par le nom de la machine et le nom de l’instance SQL où se trouve la base de données avec laquelle on veut travailler.

Paramétrage des modules

Pour pouvoir affecter les modules à une licence, il faut au préalable les décrire via le bouton « Gestion des modules ».

Un module est décrit par un code et un libellé.
C’est le code qui sera transporté dans la licence, il doit donc être identique au code utilisé dans votre application.

Paramétrage des activités

Pour pouvoir affecter la (ou les) activité à une licence, il faut au préalable les décrire via le bouton « Gestion des activités ».

Une activité est décrite par un code et un libellé.
C’est le code qui sera transporté dans la licence, il doit donc être identique au code utilisé dans votre application.

Paramétrage des volumétries

Pour pouvoir affecter la (ou les) volumétries autorisés sur une licence, il faut au préalable décrire les natures de volumétries disponibles pour l’application via le bouton « Gestion des volumétries ».

Une nature de volumétrie est décrite par un code et un libellé.
C’est le code qui sera transporté dans la licence, il doit donc être identique au code utilisé dans votre application.

Récupération du paramétrage saisi sur un autre poste

Les éléments paramétrés sont stockées dans des fichiers présents à côté de l’exe. Il est donc possible de les copier d’un poste à un autre pour éviter de devoir tout ressaisir via l’application même si le paramétrage a été fait pour un autre logiciel Isagri.

  • Sources de données = datasources.xml

  • Modules = [RadicalAppli].Modules.xml

  • Activités = activities.xml

  • Volumétries = [RadicalAppli].Volumes.xml

Si le paramétrage a été fait pour un autre logiciel Isagri, il suffit de renommer le fichier des modules et celui des volumétries en remplaçant le [RadicalAppli].

Créer une licence

L’application permet de créer une licence pour un domaine existant. Jusqu’à la 7.2.0 de l’application, on permettait aussi de créer une licence dans un nouveau domaine.
Il est préconisé d’utiliser l’option « Nouvelle licence dans un domaine existant » pour éviter de consommer des domaines utilisateurs. Une plage de n° de domaine a été réservée à cet effet et ces domaines sont flagués comme « domaine de test ».

Créer une nouvelle licence dans un domaine existant

Cliquer sur le bouton « Nouvelle licence dans un domaine existant »

Renseigner les informations générales de la licence

Renseigner un identifiant entre 1 et 99 (inclus) = Plage réservée pour les domaines de test.

La langue est obligatoire.
Avant la V7.2.0, la langue était une zone de saisie et elle devait être saisie en minuscule.

  • Nom domaine

  • Identifiant client responsable

Enregistrement

L’enregistrement de la licence peut se faire dans un fichier (.lic) ou directement dans une base de données.

Lors de l’enregistrement dans une base de données, le domaine de la licence doit déjà exister dans la base de données.
Cette option permet d’enregistrer les licences des agriculteurs en mode partenaire dans la base de l’organisme qui contient déjà le domaine et la licence de l’organisme

Créer une nouvelle licence dans un nouveau domaine (obsolète à partir de la 7.2.0)

Cliquer sur le bouton « Nouvelle licence dans un nouveau domaine »

Un nouvel identifiant de domaine unique est généré via un appel à un service DRI (comme lors de la création d’un domaine via IsaGestLic).

Les étapes de création de la licence sont les mêmes que lors de la création d’une licence dans un domaine existant (décrit ci-dessus).

La différence est que l’on doit saisir le nom du domaine et que l’identifiant du domaine n’est pas saisissable.

Ouvrir une licence existante

L’application permet d’ouvrir licence à partir d’un fichier ou directement à partir d’une licence installée dans une base de données via le bouton « Ouvrir une licence depuis… ».

Cette fonctionnalité permet de voir le contenu d’une licence interne ou client.
S’il s’agit d’une licence client, aucune modification n’est possible.
S’il s’agit d’une licence interne, les modules, activités et volumétries sont totalement modifiables. Concernant les informations générales, certaines données ne peuvent être modifiées.

Pour enregistrer les modifications, il suffit d’utiliser le bouton « disquette » et l’enregistrement se fera alors directement dans la licence (dans le fichier si ouverture depuis un fichier, dans la base de données sinon)

L’onglet « Détail » permet de voir le contenu saisi dans les autres onglets mais il permet aussi de voir d’autres informations enregistrées qui peuvent être utile lors d’une maintenance.

Si la licence n’a pas été activée, affichage de la date limitée pour activée.

Installer une licence existante en base

L’application permet d’installer une licence directement dans une base de données via le bouton « Installation licence en Base ».

Au clic sur le bouton, il vous sera demandé de sélectionner le fichier lic à installer puis de sélectionner la base de données de destination.

Lors de l’enregistrement de la licence dans la base de données, une nouvelle licence est créée si elle n’existe pas déjà. Si la licence existe déjà, elle est mise à jour. Cela permet de mettre à jour une licence qui ne pointait pas sur le bon domaine.
IsaGenLic en profite pour mettre à jour le nom du domaine dans la base. Le domaine de la licence doit déjà exister dans la base de données.

Créer une licence pour un agriculteur en mode partenaire (AMP)

Rappel

Un AMP est un agriculteur en mode partenaire avec un organisme. Il peut saisir lui-même ses données dans le logiciel mais ses données sont stockées dans le domaine de l’organisme.

Un AMP possède sa propre licence qui est enregistrée dans le domaine de l’organisme.

Pour utiliser le logiciel, l’AMP doit se connecter avec son Identifiant Client et son login/mot de passe.

Création de la licence

Renseigner les différentes infos de la licence comme décrit dans le paragraphe Créer une nouvelle licence dans un domaine existant.

La licence de l’AMP est une licence de type « Individuel ».

Elle doit avoir comme « Identifiant domaine » l’identifiant domaine de l’organisme et son « Identifiant client responsable » doit être l’identifiant client de l’organisme.

Par contre elle doit avoir un « Identifiant Licence » et un « Identifiant Client » différent de l’organisme.

Enregistrement de la licence dans la base de l’organisme

Domaine de l’organisme existant en base de donnée

Si le domaine de l’organisme a déjà été initialisé et que vous n’avez pas besoin de réutiliser plus tard cette licence AMP, il suffit d’enregistrer la licence directement dans la base de données de l’organisme via le bouton « Enregistrer en base de donnée » ou via la disquette en sélectionnant le radio-bouton « Base de données » dans la modale qui s’ouvre.

Il vous sera alors demander de sélectionner la base de données de destination.

Pour rappel, l’enregistrement direct en base de données ne peut se faire que si le domaine de la licence existe déjà dans la base.

Domaine de l’organisme non existant en base de donnée

Si le domaine de l’organisme n’existe pas encore en base de données ou que vous avez besoin de garder cette licence AMP pour la réutiliser, il suffit d’enregistrer la licence dans un fichier via le bouton « Enregistrer dans un fichier » ou via la disquette en sélectionnant le radio-bouton « Fichier » dans la modale qui s’ouvre.

Lorsque le domaine de l’organisme aura été initialisé, il suffira d’enregistrer la licence AMP comme expliqué dans le paragraphe Installer une licence existante en base
